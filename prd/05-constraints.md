# Constraints

## Technical Constraints

- Must be built with **Next.js** (App Router) and **TypeScript** — no switching frameworks mid-project
- Styling must use **Tailwind CSS** only — no additional CSS libraries (e.g., no Bootstrap, no MUI)
- Must be **fully responsive** — works on mobile, tablet, and desktop
- Must achieve a **Lighthouse score of 90+** on Performance, Accessibility, and SEO

## Time & Scope Constraints

- MVP should be completable in a **focused solo sprint**
- No backend or database required for MVP — keep it static/JAMstack
- Contact form can use a third-party service (e.g., Formspree, EmailJS) to avoid building an API
- clicking on contact me and sending me on clicking email take the user to its email section with some boiler plate code for gmail with gmail id

## Design Constraints

- Keep design **clean and minimal** — avoid over-engineering the UI . Avoid the neon colors.
- Use a **consistent color palette** use only one dark shaded color with one font color of light dark grey like . don't use more colors as it looks like AI generated 
- All content must be **readable** — sufficient contrast ratios (WCAG AA minimum)
- use the animations and slide left right on scrolling down like effects
- use the dark color shades like dark grey smokey black shades etc 

## Content Constraints

- Only include **real projects** — no placeholder or fake project entries
- Resume/CV must be **kept up to date** before launch
- No copyrighted images — use personal photos, free stock, or SVG illustrations only

## Deployment Constraints

- Must be hosted on **Vercel free tier** (no paid infrastructure for MVP)
- Domain setup is optional for MVP but recommended before sharing publicly
